How to Redeploy on Vercel

Quickly re-trigger a build from any existing deployment in your production or preview history. Follow this editorial guide to master the workflow.

1

Navigation

Go to the Deployments tab in the top navigation menu.

2

Locate

Find your latest deployment or the specific commit you wish to re-run.

3

Actions

Click the more_vert icon or Actions button next to the deployment.

4

Confirm

Select Redeploy from the dropdown menu to initiate the build.

info Pro Tip

Redeploying does not create a new commit, but it does re-trigger your build pipeline, which is useful for clearing cache or updating environment variables.
